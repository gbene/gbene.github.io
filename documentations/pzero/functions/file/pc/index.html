Import point cloud data

The Import point cloud data function can be used to import point clouds files in different formats such as: .txt, .csv, .xyz, .asc, .ply, .las and .laz.

When the import action is clicked the data preview interface is presented. This window is divided in:

  1. File selection.
  2. File import options.
  3. Preview, assign and confirm buttons.
  4. Preview panel

File selection

In the file selection section the file path can be copied or directly selected using the button. If clicked the file explorer menu is opened and the imported file can be selected.

NOTE: When importing the first 50 lines of the file are read to avoid long wait times for large datasets

After choosing the file the assign dialogue is automatically opened.

Read import options

Once imported the following import options can be defined:

  1. Start from column: column index from which the import starts.
  2. End on column: column index on which the import ends.
  3. Header line: line in which the data header is contained.
  4. Start from line: row index from which the import starts.
  5. End on line: row index on which the import ends.
  6. Separator: type of separator used in the input data file. The following separators are supported:
    • Space
    • Tab
    • Comma
    • Semicolumn
TIP: To import the whole file (all of the rows) set the end row index to -1.

Preview, assign and confirm buttons

Preview

The preview function can be used to visualize the input data. When clicked the preview panel will be populated with the first 50 data lines present in the file following the chosen import options.

Assign

A specific property can be assigned for every column of the imported dataset. This is accomplished using the assign dialogue accessed through the assign button or automatically opened when importing for the first time a file.

The dialogue is composed of three columns:

  1. Column name: The column name in the input dataset
  2. Property: The assigned property name used when importing the data in PZero.
  3. Custom property name: Custom property name.

A series of options are possible:

  1. For common properties such as XYZ, RGB or Intensity values, PZero tries to automatically match and set the column to an appropriate property name. To manually set a property select the appropriate name from the menu in the Properties column.
NOTE: If there is no match or the default name is considered fitting the property name is/can be left untouched (As is).
  1. For custom properties names the User defined option can be selected. This activates the text box in the corresponding row and Property name column.
NOTE: The user_ prefix is always appended to the input name (custom_data $\rightarrow$ user_ custom_data)
  1. To ignore a property the N.a. name can be set. In this case the column(s) corresponding to the N.a. name will not be imported.
BE AWARE: N.a. column(s) will not be imported even if they are included in the selected column range (start-end index)

When done the Ok button can be clicked and the options will be saved.

NOTE: The changes will not be visible in the preview table but the assign window can be opened to view the changes made.

Confirm

Once everything is set the data can be imported in PZero using the selected options by clicking the Ok button. The imported object will be visible in the DEMs and DOMs tab in the project explorer.
